Compatibility report

Tested in:

  • Google Chrome
  • Safari
  • Firefox

Tested Print Style:

We tested the print style in Google Chrome, Safari, and Firefox by selecting the print option in each browser. We were able to identify some small differences.

For example, by examing the printing style in the 'return policy' section, we found that 'widows' and 'orphans' are not supported by Firefox, and although Chrome and Safari support 'widows' and 'orphans', the number of lines left in a new page are different due to different default layout choices in each browser.

We also noticed that the default page headers are different in each brower, for instance, Google Chrome includes the time we printed the page, but we have the choice to ask the browsers to not include headers and footers when printing.

Tested Different Screen Sizes:

We tested different sizes by directly changing the height and the width of the browser window and also in the browser's developer mode.

All of the elements incluing images can change proportionally as the website size changes since we utilized relative sizes in the CSS file.

When we changed the size, the website worked properly in all of the three browsers.

In general, we tested print style and different sizes in Chrome, Firefox, and Safari. All of the html and css constructs work except the occasion for 'widows' and 'orphans' in all three browsers.
